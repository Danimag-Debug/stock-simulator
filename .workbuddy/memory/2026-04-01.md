## 推荐不更新 Bug 根因总结（第5-7次修复 = 最终修复）
**真正根因**：v5.0 新增了 `market_regime`、`risk_tags`、`opportunity_tags`、`updated_at` 等列，但 Railway 上的 SQLite 数据库文件是在 v5.0 之前创建的，这些列从未被成功添加。
1. `init_db()` 中有 ALTER TABLE 迁移逻辑，但用 `try/except pass` 静默吞掉了异常
2. `save_suggestions()` 的 INSERT 语句硬编码了这些列，表中不存在就抛 OperationalError
3. `init_db()` 第207行 `CREATE INDEX ON suggestions(updated_at)` 也会因列不存在而崩溃，导致整个 init_db 失败
4. `load_suggestions()` 的 `ORDER BY updated_at DESC` 在旧表上崩溃
5. **SQLite ALTER TABLE ADD COLUMN 不支持 `DEFAULT CURRENT_TIMESTAMP`**（非常量默认值），导致 updated_at 迁移失败
6. 扫描线程 catch 异常后设置 `last_error`，`summary` 从未被设置
6. 前端轮询 `running=false` 时，既无 summary 也无 skip_reason，走到 else 分支显示"推荐未变化"

**修复（53bab80 + 50756c0）**：
1. `init_db()` 迁移改为先 PRAGMA table_info 检查列是否存在，迁移列表加入 updated_at
2. 索引创建加 try/except 容错，列不存在不崩溃
3. `save_suggestions()` 动态构建 INSERT 语句，只包含表中实际存在的列
4. `load_suggestions()` 动态生成 ORDER BY（updated_at 存在才排序）
5. `get_suggestions_updated_at()` 加列检查 + 异常容错
6. 本地验证：17列旧数据库 → 迁移后25列，扫描/保存/读取全部正常

## 大盘环境判断模块 v2.0 重写
**问题**：大盘环境一直显示"获取数据中"、上证指数固定3000点
**根因**：`market_regime_analyzer` 依赖 Tushare Pro 的 `pro.daily()` 接口获取上证指数历史K线，但项目配置的是免费 Tushare Token（没有 Pro 权限），`set_tushare()` 从未成功调用，`_ts` 始终为 None → 返回默认值
**修复**：重写 `market_regime_analyzer.py`，改用东方财富免费 API `push2his.eastmoney.com` 获取上证指数 120 天日K线，完全独立运行，不再依赖任何 Tushare 接口

## 手动扫描不更新 Bug 修复
**问题**：手动扫描后推荐列表不变（仍显示旧9只），因为 `run_stock_scan` 提前退出时（大盘暴跌/Tushare失败/评分门槛过高）直接 `return []`，不调用 `save_suggestions`，旧数据不清空。
**修复**：
1. `engine_db.py`: 提前退出和结果为空时都调用 `save_suggestions([])` 清空旧数据
2. `run_stock_scan` 返回值改为 dict（含 skip_reason/skip_detail/summary），不再返回裸列表
3. `app_auth.py`: `scan_status` 新增 skip_reason/skip_detail/summary 字段，定时扫描和手动扫描都适配
4. `index_auth.html`: 扫描完成后根据状态显示具体提示（成功摘要/跳过原因/错误信息）

# 2026-03-26 开发日志

## 修复的问题（第二轮）

### 1. 价格不正确
- **根因**: `score_stock`函数中存在死代码，函数结构错误导致代码混乱
- **修复**: 
  - 将简化版评分重命名为`score_stock_simple`
  - 创建完整版`score_stock`，优先使用历史数据（MA/MACD/RSI/量比），失败时自动降级到简化版
  - 清理了`score_stock_simple`函数末尾的死代码（第423-509行）

### 2. 持仓价格未更新
- **根因**: `get_portfolio_snapshot`函数使用成本价代替实时价格
- **修复**: 在持仓快照中批量调用`ts.get_realtime_quotes()`获取实时价格

### 3. 详情按钮无响应
- **根因**: 在HTML属性中嵌入JSON字符串，特殊字符导致解析失败
- **修复**: 改用`data-*`属性存储数据 + `addEventListener`绑定事件

### 4. 手动扫描无反应
- **根因**: 按钮调用`manualScan()`但其中用的是不带Token的`fetch`，且等待时间只有1分钟
- **修复**: 
  - `manualScan`改为调用`triggerScan`
  - 按钮改为调用`triggerScan`
  - `triggerScan`增加5秒轮询机制，自动检测扫描完成并刷新推荐

### 5. Tushare导入路径问题
- **根因**: 使用相对导入`from .tushare_config`，在某些运行环境下不可用
- **修复**: 改为绝对导入，先将`__file__`目录加入`sys.path`

## 修复的问题（第三轮）

### 推荐股票展示价格为旧价格（扫描时缓存价格）
- **根因**: `load_suggestions()` 直接读取数据库中保存的价格，而数据库是每30分钟扫描时写入的，页面刷新时不会更新价格
- **修复**: 在 `load_suggestions()` 函数内，返回前调用 `ts.get_realtime_quotes()` 批量获取推荐股票的实时价格，用实时价格覆盖 `current_price`、`change_pct` 和 `buy_price`
- **降级保护**: Tushare 不可用时返回数据库缓存价格，不影响功能
- **验证**: 本地测试成功，7只股票实时价格已刷新（002787 华源控股 12.76元 +4.76%）

## 修复的问题（第四轮）

### 手动扫描/刷新推荐按钮点击后无反应
- **根因1（后端竞态）**: `scan/trigger` API 先 `Thread.start()` 再 return，前端拿到 success 立刻轮询 `/scan/status` 时，子线程还没来得及将 `running` 设为 `True`，导致轮询误判"扫描已完成"，立刻 `loadSuggestions()` 拿到的还是旧数据
- **修复1**: 先同步 `scan_status["running"] = True`，再 `Thread.start()`，确保 API 返回时状态已更新
- **根因2（前端无反馈）**: 按钮点击后无任何 loading 状态，用户无法判断是否有反应；刷新推荐直接调 `loadSuggestions()` 无任何视觉变化
- **修复2**:
  - 手动扫描按钮：触发后立即禁用+显示"扫描中..."，grid 显示 loading 占位，完成/超时后恢复
  - 新增 `refreshSuggestions()`：显示"正在获取实时价格..."占位，完成后 toast 提示，失败恢复按钮
  - 渲染逻辑抽离为 `renderSuggestions()` 函数供多处复用

## 修复的问题（第五轮，2026-03-27）

### 推荐股票每次扫描不变化 + 重复股票
- **根因1（核心）**: Tushare `pro.daily()` 历史日线接口需积分权限，当前账号没有。`score_stock` 完整版永远失败，降级到旧 `score_stock_simple`。旧简化版只看涨幅+价格+板块，所有 0-5% 涨幅+5-50元价格的股票都得 80 分，毫无区分度，每次推荐几乎一样
- **根因2（重复）**: `_build_all_market_codes()` 存在重叠范围（002xxx 被计算两次），导致同一只股票在推荐里出现两次
- **修复**:
  - `score_stock_simple` 重构为5维度评分：涨幅温和/回调/K线日内位置(low-high)/量能换手率/成交额活跃度，评分区间有区分度（70-100 各不同）
  - `get_stock_list` 额外保留 `open_price/high_price/low_price` 传入评分
  - `run_stock_scan` 增加候选池去重 + 按成交额排序 + 评分同分时随机扰动±3分
  - `save_suggestions` 写入前对 `stock_code` 去重，三重保险
